# Algorithm Pattern Templates

Memorize these. They're the lego blocks of every Medium problem.

When you see the **signal** in a problem, you immediately know the **template**.
That's how you go from "I know what to do" to "I can type it in 5 minutes".

## Pattern → Signal map

| Pattern | Signal in problem |
|---|---|
| **BFS** | Shortest path, level-by-level, unweighted graph |
| **DFS** | All paths, connected components, tree traversal |
| **Backtracking** | All subsets / permutations / combinations |
| **Sliding Window** | Subarray / substring with constraint |
| **Two Pointers** | Sorted array + pair / palindrome / cycle in linked list |
| **Binary Search** | Sorted input, or "minimize / maximize ___" answer |
| **Topological Sort** | Dependencies, ordering, prerequisites |
| **Union Find** | Connected components, dynamic merging |
| **Trie** | Prefix lookup, autocomplete, word dictionary |
