# Step 01 — Restate the Problem

## 🎯 Purpose
Confirm out loud that you understand what the interviewer asked.

## 🧠 What interviewer is testing
- Active listening
- Whether you'll start solving the WRONG problem

## 🇬🇧 English phrases (memorize verbatim)

```
"Let me make sure I understand the problem correctly."
"So we are given ___, and we need to return ___. Is that right?"
"Just to confirm, the goal is to find ___ such that ___."
```

## 📚 Required knowledge

**Restate in your OWN words. Do NOT parrot the problem verbatim.**

Pattern:
> "We are given **[input description]**. We need to return **[output description]** such that **[constraint]**."

Example — Two Sum:
> "We are given a list of integers and a target. We need to return the two indices of the numbers that add up to the target."

## ⚠️ Common mistakes

- Skipping this step → solving the wrong problem
- Parroting verbatim → shows no understanding
- Restating, but not pausing for confirmation

## 🔁 Drill

Restate each in ONE sentence (out loud, in English):

1. **Valid Parentheses**: Given a string of brackets, decide if they are balanced.
2. **Reverse Linked List**: Given the head of a list, return the head of the reversed list.
3. **Number of Islands**: Given a 2D grid of '1' and '0', count the number of connected groups of '1'.

After each, say: *"Is that right?"*
