## 🟦 PLAN (5 min)

### 7. Brute force first

> **You:** *"Brute force: for each index i, find max height to the left and max height to the right.
> Water at position i = min(left_max, right_max) - height[i], if positive.
> That's O of n squared time, O of one space."*

### 8. Inefficiency ⭐

> **You:** *"We re-scan the array for left_max and right_max every iteration.
> We're doing the same scans repeatedly."*

### 9. Optimization 1 — DP with two arrays

> **You:** *"First optimization — precompute left_max and right_max in two passes.
> O of n time, O of n space."*

### 10. Optimization 2 — Two Pointers ⭐⭐⭐

> **You:** *"Even better — two pointers from both ends.
>
> Key insight: at each step, whichever side is SHORTER limits the water height,
> regardless of what's on the other side.
> If height[left] < height[right], we KNOW the right side has at least height[right],
> which is taller than left. So water at left is determined by left_max alone.
> We process left and move forward.
> Same logic when right is shorter.*
>
> *This gives O of n time, O of one space."*

### 11. Visual — why two pointers works

```text
height = [3, 1, 2, 5, 0, 4]
          ^                 ^
          L                 R

left_max = 3, right_max = 4
height[L]=3 < height[R]=4

→ Water at L is determined by left_max (3),
  because right_max ≥ height[R]=4 > height[L]=3.
→ left_max already covers it. Move L forward.

         [3, 1, 2, 5, 0, 4]
             ^           ^
             L           R

height[L]=1 < left_max=3
→ Water trapped here = left_max - height[L] = 3 - 1 = 2
→ Move L forward.
```

### 12. Complexity upfront

> **You:** *"Two pointers gives O of n time, O of one extra space. Optimal."*

### 13. Buy-in

> **You:** *"Sound good before I code?"*
> **Interviewer:** *"Go ahead."*

## 🟦 CODE (12 min)

### 14.0 Quick check

> **You:** *"Just the function, right? No class wrapper?"*
> **Interviewer:** *"Just the function."*

### 14.1 Approach (recap before typing)

> **You:** *"I'll use two pointers — left at start, right at end.
> Track left_max and right_max.
> Each step, advance the smaller side.
> If current height is less than that side's max, water is trapped — add to total."*

### 14.2 The code

```python
def trap(height: list[int]) -> int:
    """Return total water trapped between bars."""
    if not height:
        return 0

    left, right = 0, len(height) - 1
    left_max = right_max = 0
    total = 0

    while left < right:
        if height[left] < height[right]:
            # left side is shorter — water bounded by left_max
            if height[left] >= left_max:
                left_max = height[left]
            else:
                total += left_max - height[left]
            left += 1
        else:
            # right side is shorter or equal — water bounded by right_max
            if height[right] >= right_max:
                right_max = height[right]
            else:
                total += right_max - height[right]
            right -= 1

    return total
```

### 14.3 What to say while typing

- *"Empty check first — return 0."*
- *"left and right pointers at the ends. left_max and right_max start at 0."*
- *"Loop while left less than right."*
- *"Compare heights — work on the smaller side because it limits water there."*
- *"If current height is at least the running max, update the max — no water here."*
- *"Otherwise, water is the difference. Add to total."*
- *"Move the pointer inward."*

### 14.4 Naming choices

- `left_max` / `right_max` — running max from each side
- `total` — accumulated water (instead of `result` for clarity)
- `left` / `right` — standard two-pointer naming

### 16. Edge cases

> **You:** *"Edge cases:*
> *- Empty array → return 0 (handled by guard).*
> *- Single bar → loop doesn't run (left == right), return 0.*
> *- Two bars → no space between, return 0.*
> *- Monotonically increasing [1,2,3,4] → no water trapped, total stays 0.*

### 17. Off-by-one

> **You:** *"Loop is `while left < right`. When they meet, no more space. Pointers always move inward, no infinite loop risk."*

## 📝 Final Code (copy-paste ready)

### Two Pointers — recommended

```python
def trap(height: list[int]) -> int:
    """Return total water trapped between bars."""
    if not height:
        return 0

    left, right = 0, len(height) - 1
    left_max = right_max = 0
    total = 0

    while left < right:
        if height[left] < height[right]:
            if height[left] >= left_max:
                left_max = height[left]
            else:
                total += left_max - height[left]
            left += 1
        else:
            if height[right] >= right_max:
                right_max = height[right]
            else:
                total += right_max - height[right]
            right -= 1

    return total
```

**Complexity:**

- Time: **O(n)** — single pass with two pointers
- Space: **O(1)** — constant extra space

### DP alternative — easier to explain, more memory

```python
def trap_dp(height: list[int]) -> int:
    n = len(height)
    if n == 0:
        return 0

    left_max = [0] * n
    right_max = [0] * n

    left_max[0] = height[0]
    for i in range(1, n):
        left_max[i] = max(left_max[i - 1], height[i])

    right_max[n - 1] = height[n - 1]
    for i in range(n - 2, -1, -1):
        right_max[i] = max(right_max[i + 1], height[i])

    return sum(min(left_max[i], right_max[i]) - height[i] for i in range(n))
```

**Complexity:**

- Time: **O(n)** — three passes
- Space: **O(n)** — two arrays

> 💡 **In the interview**: present BOTH if there's time. Two pointers is the impressive answer, DP is easier to explain.

## 🧠 Key insight to memorize

> **"The shorter side limits the water height, regardless of the other side."**
>
> If height[left] < height[right], we don't need to know exactly how tall the right side is — we just know it's at least taller than left, which means left_max is what bounds water on the left.

This is the entire trick. Once you internalize it, the code writes itself.
