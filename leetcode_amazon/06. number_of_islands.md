# Number of Islands (200) — 30-min Interview Script

> **LeetCode #200 (Medium) — ⭐⭐⭐ Amazon TOP question.**
> DFS or BFS on a 2D grid. Read OUT LOUD. Total: ~22 min.

## 🎬 Setup

**Interviewer:** *"Given a 2D grid of '1' (land) and '0' (water), count the number of islands. An island is a group of '1's connected horizontally or vertically."*

> **You:** *"I have a 2D grid. Each cell is either '1' (land) or '0' (water).

### 3. Output

> **You:** *"Return a single integer — number of islands. Empty grid returns 0?"*
> **Interviewer:** *"Yes, 0."*

### 4. Constraint — can I mutate the grid?

> **You:** *"Can I modify the grid in place to mark visited cells, or should I preserve it?"*
> **Interviewer:** *"You can modify it."*

### 5. Example

> **You:** *"Could I get an example?"*
>
> **Interviewer:**

```text
[
  ["1","1","0","0","0"],
```

> **You:** *"OK — top-left island of four 1's, middle 1 alone, bottom-right pair. Total 3."*

## 🟦 PLAN (4 min)

### 7. Approach — DFS or BFS scan

> **You:** *"Iterate through every cell. When I find a '1' that hasn't been visited,
> increment count, then run DFS (or BFS) to mark every connected '1' as visited.
> Each connected group is one island."*

### 8. Why DFS works here

> **You:** *"DFS is natural because we just want to MARK all connected cells — order doesn't matter.
> BFS works equally well. I'll go with DFS because the recursion is cleaner."*

### 9. Marking visited — mutate the grid

> **You:** *"To save memory, I'll mark visited cells by mutating the grid — change '1' to '0' (or '#').
> Alternative: a separate visited set, O of rows times cols extra space.
> Mutating is simpler since the interviewer said it's allowed."*

### 10. Complexity upfront

> **You:** *"Time O of rows times cols — every cell visited at most once.*
> *Space O of rows times cols in the worst case — recursion stack when the whole grid is one island."*

### 11. Buy-in

> **You:** *"Sound good before I code?"*
> **Interviewer:** *"Go ahead."*

## 🟦 CODE (10 min)

### 12.1 Approach (recap)

> **You:** *"Loop every cell. On unvisited '1', increment count and DFS to sink the island. DFS marks land as water."*

### 12.2 The code

```python
def num_islands(grid: list[list[str]]) -> int:
    """Count connected groups of '1' (4-directional)."""
    if not grid or not grid[0]:
        return 0

    rows, cols = len(grid), len(grid[0])
    count = 0

    def dfs(r: int, c: int) -> None:
        # Out of bounds or water — stop
        if r < 0 or r >= rows or c < 0 or c >= cols or grid[r][c] != "1":
            return
        # Mark visited by sinking the land
        grid[r][c] = "0"
        # Visit 4 neighbors
        dfs(r + 1, c)
        dfs(r - 1, c)
        dfs(r, c + 1)
        dfs(r, c - 1)

    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == "1":
                count += 1
                dfs(r, c)

    return count
```

### 12.3 What to say while typing

- *"Empty grid check first."*
- *"rows and cols variables."*
- *"DFS helper inside — bounds check first, then water check, then recurse."*
- *"Sink the land by setting it to '0' — that's our visited mark."*
- *"Outer double loop — every cell. On unvisited '1', count++ and run DFS."*

### 12.4 Naming

- `dfs` — clear
- `count` — running island count
- `grid[r][c] != "1"` — handles both bounds-checked '0' and visited cells

> 💡 **Important**: comparing to string `"1"` not int `1` — input is a 2D char array.

## 🟦 COMPLEXITY (1 min)

### 18. Final

> **You:** *"Time O of m times n — each cell visited at most once. The DFS doesn't double-count because we sink land as we go.*
> *Space O of m times n in the worst case — recursion stack if the whole grid is one big island.*
>
> *Trade-off: BFS would have the same time but O of width space (BFS queue) instead of O of depth.*
> *For very large grids where recursion might overflow, BFS is safer.*
>
> *Union-Find is another option — same time, but more code."*

## 📝 Final Code (copy-paste ready)

### DFS — recommended (simpler)

```python

    """Count connected groups of '1' (4-directional)."""
    if not grid or not grid[0]:
        return 0

    rows, cols = len(grid), len(grid[0])
    count = 0

    def dfs(r: int, c: int) -> None:
        if r < 0 or r >= rows or c < 0 or c >= cols or grid[r][c] != "1":
            return
        grid[r][c] = "0"
        dfs(r + 1, c)
        dfs(r - 1, c)
        dfs(r, c + 1)
        dfs(r, c - 1)

        for c in range(cols):

                count += 1
                dfs(r, c)

```

### BFS alternative — safer for huge grids

```python
from collections import deque


def num_islands_bfs(grid: list[list[str]]) -> int:
    if not grid or not grid[0]:
        return 0

    rows, cols = len(grid), len(grid[0])
    count = 0

    for r in range(rows):
        for c in range(cols):
            if grid[r][c] != "1":
                continue
            count += 1
            queue = deque([(r, c)])
            grid[r][c] = "0"
            while queue:
                cr, cc = queue.popleft()
                for dr, dc in ((1, 0), (-1, 0), (0, 1), (0, -1)):
                    nr, nc = cr + dr, cc + dc
                    if 0 <= nr < rows and 0 <= nc < cols and grid[nr][nc] == "1":
                        grid[nr][nc] = "0"
                        queue.append((nr, nc))

    return count
```

**Complexity (both):**

- Time: **O(m × n)** — each cell visited once
- Space: **O(m × n)** worst case — recursion (DFS) or queue (BFS) when the grid is one big island

## 🧠 Pattern: Connected Components on Grid — what to remember

| When to use | Count groups of connected cells in a grid |
|---|---|
| Choice of traversal | DFS (cleaner code) or BFS (safer for huge grids) |
| Visited tracking | Mutate grid (simpler) OR separate set (preserves input) |
| Direction vectors | `[(1,0), (-1,0), (0,1), (0,-1)]` for 4-directional |

- Surrounded Regions (130) — DFS from borders
- Pacific Atlantic Water Flow (417) — DFS from two borders
