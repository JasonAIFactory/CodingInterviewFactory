## 🟦 PLAN (3 min)

### 7. Brute force

> **You:** *"Brute force: for each day i, check every later day j. profit = prices[j] - prices[i]. Track max. O of n squared."*

### 8. Inefficiency ⭐

> **You:** *"For each i, we re-scan everything later. We don't need to know all later prices — just whether ANY later price would have given us higher profit."*

### 9. Optimization — single pass tracking

> **You:** *"One-pass approach. Track the MIN PRICE seen so far. For each price, compute profit if I sold today. Update max profit.
> The trick: as we walk forward, the best buying day so far is just the cheapest price we've seen. We don't need to track all of them."*

### 10. Complexity upfront

> **You:** *"O of n time, O of one space. Single pass with two variables."*

### 11. Buy-in

> **You:** *"Sound good before I code?"*
> **Interviewer:** *"Go ahead."*

---

## 🟦 CODE (8 min)

### 12.1 Approach

> **You:** *"Walk through prices once. Keep `min_price` (cheapest so far) and `max_profit` (best deal so far)."*

### 12.2 The code

```python
def max_profit(prices: list[int]) -> int:
    """Return max profit from buying once and selling later."""
    if not prices:
        return 0

    min_price = prices[0]
    max_profit = 0

    for price in prices[1:]:
        max_profit = max(max_profit, price - min_price)
        min_price = min(min_price, price)

    return max_profit
```

### 12.3 What to say while typing

- *"Empty check, return 0."*
- *"Initialize min_price as first price, max_profit as 0."*
- *"Loop from second price."*
- *"For each price, check if selling today gives better profit. Update max_profit."*
- *"Then update min_price if current is cheaper — for future iterations."*

### 12.4 Order matters

> **You:** *"Important — I update `max_profit` BEFORE `min_price`.
> Because if I update min_price first to the current price, then `price - min_price` becomes 0 — I'd accidentally allow same-day buy/sell."*

## 📝 Final Code (copy-paste ready)

```python
def max_profit(prices: list[int]) -> int:
    """Return max profit from buying once and selling later."""
    if not prices:
        return 0

    min_price = prices[0]
    max_profit = 0

    for price in prices[1:]:
        max_profit = max(max_profit, price - min_price)
        min_price = min(min_price, price)

    return max_profit
```

**Complexity:**

- Time: **O(n)** — single pass
- Space: **O(1)** — two scalars
