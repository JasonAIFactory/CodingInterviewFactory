## 🟦 PLAN (3 min)

### 8. Brute force

> **You:** *"Brute force: try every substring, check uniqueness with a set. That's O(n cubed) — n squared substrings, each O(n) to check."*

### 9. Inefficiency ⭐

> **You:** *"We are re-checking the same characters every time we extend the substring. Most of that work was already done in the previous iteration."*

### 10. Optimization + data structure

> **You:** *"I'll use the sliding window pattern with two pointers — left and right.
> I expand right one step at a time. If the new character was already seen INSIDE the current window, I jump left past the previous occurrence.
> I store last seen index of each character in a hash map for O of one lookup."*

### 11. Complexity upfront

> **You:** *"O(n) time — each character is visited at most twice. O(min of n and alphabet size) space for the hash map."*

### 12. Buy-in ⭐

> **You:** *"Does this approach sound good before I start coding?"*
> **Interviewer:** *"Go ahead."*

## 🟦 CODE (10 min)

### 13.0 Quick check

> **You:** *"Function or class wrapper?"*
> **Interviewer:** *"Just the function."*

### 13.1 Approach (key insight) ⭐

> **You:** *"The key insight is to never re-scan.
> The window has a left and right boundary.
> Right always moves forward.
> Left only moves when we hit a repeat — and it jumps directly past the previous occurrence using the hash map.
> So both pointers go one direction only, total O of n."*

Mental model:

```text
s = "a b c a b c b b"
     ^               left=0, right=0
     a b c           valid window, len=3, best=3
           a         repeat 'a' at idx 0! left jumps to 1
       b c a         valid, len=3
             b       repeat 'b' at idx 1! left jumps to 2
         c a b       valid, len=3
               c     repeat 'c'! left jumps to 3
           a b c     valid, len=3
                 b   repeat 'b' at idx 4! left jumps to 5
               c b   valid, len=2
                   b repeat 'b' at idx 6! left jumps to 7
                 b   valid, len=1
best = 3
```

### 13.2 The code

```python
def length_of_longest_substring(s: str) -> int:
    """Return the length of the longest substring with no repeating characters."""
    last_seen: dict[str, int] = {}
    left = 0
    best = 0

    for right, ch in enumerate(s):
        if ch in last_seen and last_seen[ch] >= left:
            left = last_seen[ch] + 1
        last_seen[ch] = right
        best = max(best, right - left + 1)

    return best
```

### 13.3 What to say while typing

- *"Hash map last_seen — character to its last index."*
- *"Left pointer starts at 0. Best length starts at 0."*
- *"Loop with enumerate — right is the index, ch is the current character."*
- *"If ch was seen and that index is INSIDE the window, jump left past it."*
- *"Update last_seen for ch."*
- *"Update best with current window length, right minus left plus one."*

### 13.4 Naming choices

- `last_seen` — clear: "last index where I saw this character"
- `left`, `right` — standard for two-pointer / sliding window
- `best` — running max; `result` or `max_len` also OK
- `ch` — single character, short OK in tight loop

> 💡 **Important condition**: `last_seen[ch] >= left` — only jump if the previous occurrence is INSIDE the current window. Otherwise it was BEFORE left and we can ignore it.

### 15. Dry run

> **You:** *"With s = 'abcabcbb':*
> *right=0, ch='a', not seen, store {'a':0}, best=1.*
> *right=1, ch='b', not seen, store {'b':1}, best=2.*
> *right=2, ch='c', not seen, store {'c':2}, best=3.*
> *right=3, ch='a', seen at 0 which is >= left=0, jump left to 1, store {'a':3}, best stays 3.*
> *... and so on. Final best = 3. Correct."*

### 16. Edge cases

> **You:** *"Edge cases:*
> *- Empty string '' → loop doesn't run, return 0.*
> *- Single char 'a' → best becomes 1.*
> *- All same 'bbbb' → keeps jumping left forward, window stays size 1, best=1.*
> *- All unique 'abcde' → never jumps, best=5.*
> *- Spaces and digits — treated as any character, no special handling."*

### 17. Off-by-one

> **You:** *"Window length is right minus left plus one — that's the inclusive length, easy to forget the plus one."*

## 📝 Final Code (copy-paste ready)

```python
def length_of_longest_substring(s: str) -> int:
    """Return the length of the longest substring with no repeating characters."""
    last_seen: dict[str, int] = {}
    left = 0
    best = 0

    for right, ch in enumerate(s):
        if ch in last_seen and last_seen[ch] >= left:
            left = last_seen[ch] + 1
        last_seen[ch] = right
        best = max(best, right - left + 1)

    return best
```

**Complexity:**

- Time: **O(n)** — each char processed once
- Space: **O(min(n, alphabet size))** — typically O(1) for ASCII

## 🧠 Pattern: Sliding Window — what to remember

| When to use | "Longest / shortest / max / min substring with constraint" |
|---|---|
| Data structure | Two pointers + hash map |
| Time | O(n) — each element visited at most twice |
| Space | O(distinct elements in window) |
| Key trick | Left only moves forward; never restarts |
