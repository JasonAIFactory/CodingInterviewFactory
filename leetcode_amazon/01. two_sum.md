## 🟦 PLAN (3 min)

### 8. Brute force

> **You:** *"Brute force: for each i, check every j after it. O(n squared) time, O(1) space."*

### 9. Inefficiency ⭐

> **You:** *"We re-scan the array for every i. Same lookup over and over."*

### 10. Optimization + data structure

> **You:** *"I'll use a hash map from value to index. For each num, I check if its complement is in the map. Hash map gives O(one) lookup."*

### 11. Complexity upfront

> **You:** *"O(n) time, O(n) space. We trade space for time."*

### 12. Buy-in ⭐ - 동의

> **You:** *"Does this sound good before I start coding?"*
> **Interviewer:** *"Go ahead."*

### 13.1 Approach (the key insight) ⭐

Before typing, briefly say the THINKING out loud:

> **You:** *"The key insight is that for each number,
> I need to know if I have already seen its complement —
> the value that pairs with it to make the target.*
>
> *Brute force would re-scan the array for each i.
> But if I store every number I've seen in a hash map,
> I can do that lookup in O of one.*
>
> *So I walk through the array left to right.
> For each number, I ask: 'is target minus num already in my map?'
> If yes, I have my pair. If no, I store this number for future lookups.*
>
> *One pass, O of n time."*

Mental model in one image:

```text
nums =   [2,  7, 11, 15],  target = 9

i=0  num=2   complement=7   seen={}            store {2:0}
i=1  num=7   complement=2   seen={2:0}  found! return [0, 1]
```

### 13.2 The code

```python
def two_sum(nums: list[int], target: int) -> list[int]:
    """Return indices of two numbers that sum to target."""
    seen: dict[int, int] = {}

    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i

    return []
```

### 13.3 What to say while typing (concise)

- *"Empty hash map called seen — value to index."*
- *"Loop with enumerate to get index and value together."*
- *"Compute complement equals target minus num."*
- *"If complement is already in seen, return both indices."*
- *"Otherwise store the current number with its index."*
- *"Return empty list at the end as a safety net."*

### 13.4 Naming choices (in case interviewer asks)

- `seen` — Python convention for "values I've already encountered"
- `complement` — exactly what it represents (`target - num`)
- `i`, `num` — short OK because they live inside one tight loop
- Type hints (`list[int]`, `dict[int, int]`) make intent clear without long names

## 📝 Final Code (copy-paste ready)

```python
def two_sum(nums: list[int], target: int) -> list[int]:
    """Return indices of two numbers that sum to target."""
    seen: dict[int, int] = {}

    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i

    return []
```

**Complexity:**

- Time: **O(n)** — single pass
- Space: **O(n)** — hash map of up to n entries
