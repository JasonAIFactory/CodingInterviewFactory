# Container With Most Water (11) — 25-min Interview Script

> **LeetCode #11 (Medium) — Amazon classic.**
> Two pointers (opposite ends) — always move the SHORTER side. Read OUT LOUD. Total: ~20 min.

---

## 🎬 Setup

**Interviewer:** *"Given n non-negative integers where each represents a vertical line of that height, find two lines that together with the x-axis form a container holding the most water. Return the max water area."*

(That's all. Everything else, you ASK.)

---

## 🟦 CLARIFY (3 min)

### 1. Restate

> **You:** *"I have heights of vertical lines. I pick two lines, treat them as walls of a container. Water is bounded by the SHORTER line. Width is the distance between them. I return the maximum area possible."*
> **Interviewer:** *"Yes."*

### 2. Input

> **You:** *"Max length? Can heights be zero?"*
> **Interviewer:** *"Length up to 10 to the 5. Heights non-negative, up to 10 to the 4."*

### 3. Output

> **You:** *"Return a single integer — the max area."*
> **Interviewer:** *"Yes."*

### 4. Definition

> **You:** *"Area = min of two heights times distance between them, right? Two lines must be different positions?"*
> **Interviewer:** *"Correct on both."*

### 5. Example

> **You:** *"Could I get an example?"*
>
> **Interviewer:** *"`[1,8,6,2,5,4,8,3,7]` → output 49. Pick index 1 (height 8) and index 8 (height 7). Width = 7, height = min(8,7) = 7, area = 49."*
>
> **You:** *"OK — width times shorter height."*

### 6. My own example

> **You:** *"Let me try `[1,1]` — area = min(1,1) * 1 = 1. Right?"*
> **Interviewer:** *"Yes."*

---

## 🟦 PLAN (3 min)

### 7. Brute force

> **You:** *"Brute force: try every pair (i, j). Compute area, track max. O of n squared."*

### 8. Inefficiency ⭐

> **You:** *"We're checking many pairs that can't possibly be the answer.
> If we have a wider container with smaller heights, we don't always need to keep checking narrower ones with the same heights."*

### 9. Optimization — Two Pointers ⭐⭐⭐

> **You:** *"Two pointers from both ends — left at start, right at end.
> Compute current area. Then move the SHORTER side inward.*
>
> *Why move the shorter side? Because moving the taller side can only DECREASE area —
> width decreases AND height stays bounded by the shorter line.
> But moving the shorter side might find a taller line that increases the area
> despite the smaller width."*

### 10. Visual — why move the shorter side

```text
[1, 8, 6, 2, 5, 4, 8, 3, 7]
 ^                       ^
 L (h=1)                 R (h=7)

Area = min(1, 7) * 8 = 8
Bounded by L (height 1).

If I move R inward (taller side):
- Width drops to 7
- Height still bounded by L=1
- Area = 1 * 7 = 7 — WORSE

If I move L inward (shorter side):
- Width drops to 7
- But height could JUMP to 8
- Area = min(8, 7) * 7 = 49 — BETTER!
```

> **You:** *"So always move the shorter side. That's the key insight."*

### 11. Complexity upfront

> **You:** *"O of n time — pointers move toward each other, n total iterations.*
> *O of one space."*

### 12. Buy-in

> **You:** *"Sound good before I code?"*
> **Interviewer:** *"Go ahead."*

---

## 🟦 CODE (8 min)

### 13.1 Approach (recap)

> **You:** *"Two pointers, compute area each step, move whichever side is shorter."*

### 13.2 The code

```python
def max_area(height: list[int]) -> int:
    """Find max water area from two lines."""
    left, right = 0, len(height) - 1
    best = 0

    while left < right:
        width = right - left
        area = min(height[left], height[right]) * width
        best = max(best, area)

        if height[left] < height[right]:
            left += 1
        else:
            right -= 1

    return best
```

### 13.3 What to say while typing

- *"Two pointers at both ends. best starts at 0."*
- *"Loop while left less than right."*
- *"Compute width = right - left, area = min of heights times width."*
- *"Update best with max."*
- *"Move the shorter side inward."*

### 13.4 Naming

- `best` — running max area; `result` or `max_area_seen` also fine
- `width` — explicit name for clarity (vs `right - left` inline)

### 16. Edge cases

> **You:** *"Edge cases:*
> *- Length 2: only one pair, area = min * 1.*
> *- All same height: each pair has area = h * (j-i). Max is widest pair = h * (n-1).*
> *- All zeros: area always 0.*
> *- Strictly increasing: shorter side is always left → keep moving left forward.*
> *- Empty array: my loop doesn't execute, returns 0 (best initial value)."*

### 17. Off-by-one

> **You:** *"Loop ends when left == right — no width left. width = right - left correctly excludes that case."*

## 🟦 COMPLEXITY (1 min)

### 18. Final

> **You:** *"Time O of n — pointers walk toward each other, total moves are n.*
> *Space O of one.*
>
> *Trade-off: brute force is O of n squared. For n = 10 to the 5, that's 10 to the 10 — too slow. Two pointers is necessary."*

## 📝 Final Code (copy-paste ready)

```python
def max_area(height: list[int]) -> int:
    """Find max water area between two lines."""
    left, right = 0, len(height) - 1
    best = 0

    while left < right:
        width = right - left
        area = min(height[left], height[right]) * width
        best = max(best, area)

        if height[left] < height[right]:
            left += 1
        else:
            right -= 1

    return best
```

**Complexity:**

- Time: **O(n)** — pointers traverse the array once
- Space: **O(1)** — only scalar variables

---

## 🧠 Key insight to memorize

> **"Always move the shorter side."**
>
> Moving the taller side can only DECREASE area (width drops, height still bounded).
> Moving the shorter side has a CHANCE of increasing area (height could jump up).

- *"Always move the shorter side because the taller side can never improve while it's the shorter one bounding the height."*
