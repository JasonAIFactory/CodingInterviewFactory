# Dynamic Programming

**O(states × transitions)** — break into overlapping subproblems and cache.

## 🎯 Essence (memorize this one line)

> *"Cache subproblem answers to avoid recomputing. Needs overlapping subproblems and optimal substructure."*

## 🧠 How it works (principle)

Decompose a big problem into smaller subproblems and CACHE the answers to avoid recomputing. Two requirements must hold:

1. **Optimal substructure** — the optimal answer for the big problem is built from optimal answers to smaller ones
2. **Overlapping subproblems** — the same small problem appears many times during recursion

Without overlap, plain recursion suffices. Without optimal substructure, DP doesn't apply (need backtracking or BFS instead).

## 🔵 Use when

- Counting / optimization with overlapping subproblems
- "Find min / max / count of ___"
- Coin Change, LIS, Edit Distance, House Robber, Word Break

## 💡 4-step approach

1. **Define state** — what does `dp[i]` mean?
2. **Recurrence** — how does `dp[i]` relate to smaller subproblems?
3. **Base case**
4. **Order of computation**

## 💡 Template (top-down memoization)

```python
from functools import lru_cache

@lru_cache(maxsize=None)
def dp(i):
    if i <= base:
        return base_value
    return f(dp(i - 1), dp(i - 2), ...)
```

## 💡 Template (bottom-up)

```python
dp = [0] * (n + 1)
dp[0] = base
for i in range(1, n + 1):
    dp[i] = f(dp[i - 1], ...)
return dp[n]
```

## 💡 Template (2D DP — Edit Distance style)

```python
dp = [[0] * (m + 1) for _ in range(n + 1)]
for i in range(n + 1):
    dp[i][0] = i
for j in range(m + 1):
    dp[0][j] = j
for i in range(1, n + 1):
    for j in range(1, m + 1):
        if a[i-1] == b[j-1]:
            dp[i][j] = dp[i-1][j-1]
        else:
            dp[i][j] = 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1])
```

## 🇬🇧 What to say

- *"I'll use DP because subproblems overlap."*
- *"Let me define the state — dp of i means ___."*
- *"Top-down with memoization is easier; bottom-up is faster and avoids stack overflow."*
